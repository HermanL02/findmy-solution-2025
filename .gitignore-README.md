## üö® Before Every Commit

```bash
# 1. Check what's being committed
git status

# 2. Verify .env is ignored
git check-ignore .env
# Should output: .env

# 3. Review changes
git diff --cached

# 4. If .env shows up, DO NOT COMMIT
# Remove it and check .gitignore
```

## üîß Quick Fixes

### If .env was accidentally staged:
```bash
git reset HEAD .env
```

### If .env was committed:
```bash
# 1. Remove from this commit
git reset --soft HEAD~1
git reset HEAD .env
git commit -c ORIG_HEAD

# 2. Change your MongoDB password immediately!
```

### Test .gitignore:
```bash
# Check specific file
git check-ignore -v .env

# Check all ignored files
git status --ignored
```
